# Sprint 1 Planning report
## Sprint Number & Dates
Sprint: Sprint 1
Duration: Week 1 and Week 2 
## Sprint Goal
The goal of Sprint 1 is to establish the project foundation by forming the project team, selecting a realistic project topic, defining the product vision and project plan, choosing appropriate technologies, and setting up project management tools and the initial product backlog in accordance with Agile and Scrum practices.
## Selected Product Backlog Items
- Define product vision and project goal
- Identify project scope and deliverables
- Set up Trello board and GitHub Repository
- Create and prioritize user stories
- Design initial system architecture
- Prepare topic presentation
## Planned Tasks / Breakdown
1. Project Topic Selection
- Select messaging application as project topic

2. Product Vision
- Draft Product Vision document
- Define problem statement, target users, value proposition, features, and vision statement

3. Project Plan
- Define project objectives, scope and deliverables
- Create sprint-based timeline aligned with course schedule
- Identify risks, testing strategy and documentation plan

4. Technology Selection
- Select frontend, backend, database and supporting tools
- Ensure technology stack meets course requirements

5. Project Management Tools Setup
- Create Trello workspace and invite team members and lecturer
- Set up product backlog and Sprint 1 board
- Create GitHub repository

6. Initial Product Backlog
- Identify key features
- Write user stories and draw use case diagram
- Prioritize backlog items

7. Presentation Preparation
- Prepare 25-minute topic presentation
- Draft two discussion questions for class interaction

## Team Capacity & Assumptions
- All team members are available throughout the sprint
- Sprint 1 focuses on planning and documentation
- No major technical implementation is expected during this sprint

## Definition of Done
- Project team is formed and roles are assigned
- Project topic is finalized and approved
- Product Vision document is completed
- Project Plan document is completed
- Technology stack is selected and documented in GitHub README
- Trello board and GitHub repository are properly set up
- Initial product backlog is created, estimated, and prioritized
- Sprint 1 presentation is prepared
- Team is ready for the Sprint Review meeting
